## Overview
A VPC is a public cloud offering that lets an enterprise establish its own private cloud-like computing environment on shared public cloud infrastructure. 

With IBM Cloud Virtual Private Cloud (VPC), you can use the UI, CLI, and API to quickly provision generation 2 virtual server instances for VPC with high network performance. VPC infrastructure contains a number of Infrastructure-as-a-Service (IaaS) offerings, including Virtual Servers for VPC

### VPC & VSI

This section walkthrough the high level overview of the Virtual Private Cloud (VPC)

- Creation of VPC with its first subnet, ACL and Security group
- Creation of second submet in a different zone
- Control VPC network traffic via Access Control Lists and Security Groups
- Creation of Virtual Server Instances (VSI)
- Creation of SSH Keys to use with virtual server instances in VPC
- Creation of Load Balancer with health checks in VPC
- Introduction of VPC Gen2 and comparison to Gen1
- Creation of VPC Gen2
- Creation of Virtual Server Instances from a template
- Introduction of VPC Autoscaler
- Creation of VPC Autoscaler
